# 납품 체크리스트

납품 전후로 아래 항목을 순서대로 확인하세요.

---

## 납품 전 — 개발자 확인

### 코드
- [ ] .env.example 파일 포함됐는지 확인
- [ ] .env 파일이 납품 파일에 포함되지 않았는지 확인
- [ ] credentials.json 파일이 납품 파일에 포함되지 않았는지 확인
- [ ] 코드 내 하드코딩된 개인정보/API 키 없는지 확인
- [ ] 야간 발송 제한(is_allowed_time) 적용됐는지 확인
- [ ] 오류 알림(error_handler) 적용됐는지 확인

### 고객 확인
- [ ] 마케팅 수신 동의 여부 확인 완료
- [ ] 요구사항 최종 확인 (서면으로 남기기)
- [ ] API 키 발급 완료 여부 확인
- [ ] 사용 OS 확인 (Windows / Mac)
- [ ] Python 설치 여부 확인

---

## 납품 시 — 화면공유 진행

- [ ] 고객 PC에서 직접 패키지 설치 확인
  ```
  pip install requests python-dotenv gspread google-auth schedule
  ```
- [ ] .env 파일 작성 같이 진행
- [ ] 실행 테스트 완료
- [ ] 정상 작동 화면 영상 저장 (분쟁 대비 증거)
- [ ] 절전모드 해제 설정 완료
- [ ] 작업 스케줄러 자동 시작 설정 완료 (원하는 경우)
- [ ] 백신 예외 처리 완료 (필요한 경우)

---

## 납품 후 — 고객 안내

- [ ] 사용설명서.md 전달
- [ ] .env 파일 보안 주의사항 안내
  - 카카오톡/이메일로 전송 금지
  - 구글드라이브/Dropbox 업로드 금지
- [ ] API 키 재발급 안내
  - 납품 완료 후 보안을 위해 API 키 재발급 권장
  - 재발급 후 .env 파일 업데이트 방법 안내
- [ ] 오류 발생 시 연락 방법 안내
- [ ] 유지보수 패키지 안내 (필요 시)

---

## API 키 재발급 체크리스트

납품 완료 후 고객에게 아래 키 재발급을 권장하세요.

| API | 재발급 방법 |
|---|---|
| Gmail 앱 비밀번호 | Google 계정 → 보안 → 앱 비밀번호 → 기존 삭제 후 재생성 |
| 솔라피 API Key | solapi.com → API Key 관리 → 재발급 |
| 네이버 API | developers.naver.com → 애플리케이션 → Secret 재발급 |
| YouTube API | Google Cloud Console → API 키 → 재생성 |
| Twitter API | developer.twitter.com → Keys and Tokens → Regenerate |
| Notion Token | notion.so/my-integrations → 재발급 |

> ⚠️ 재발급 후 .env 파일의 해당 항목을 새 키로 교체해야 합니다.
